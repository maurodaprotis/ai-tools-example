# Review Skill

Use this skill when reviewing a branch, PR, or diff.

## Rules

- Focus on bugs and regressions first
- Call out risky assumptions
- Mention testing gaps
- Keep summaries brief after findings

## Suggested Workflow

1. Read the diff
2. Identify behavioral changes
3. Look for edge cases
4. Check whether tests cover the change
5. Summarize findings by severity
